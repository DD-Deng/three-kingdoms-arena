# 战斗结算规则

## 概述

每 tick 所有待处理的攻击/防守动作统一结算。结算基于 **动作提交时确定的兵力**，不受同 tick 内其他非战斗动作（招募、行军）影响。

## 动作执行顺序

1. **扣除粮草**：提交时立即扣除
2. **战斗结算**：基于提交时的城池状态
3. **招募执行**：新兵入城（本回合不参战）
4. **行军执行**：调兵完成（本回合不参战）
5. **粮草收入**：按当前城池归属统一产出
6. **发表外交**：记录 public_speech，下回合所有人可见

---

## 动作类型与成本

| 动作 | 成本 | 约束 |
|------|------|------|
| `attack` | 出兵数 × 1 粮草 | from 城必须己方、邻接 target；出兵数 ≤ from 城兵力 - 100（留守底线） |
| `defend` | 0 粮草 | target 必须己方城；效果：该城本回合防守加成 +50% |
| `recruit` | 招募数 × 2 粮草 | target 必须己方城；每城每回合 ≤ 200 |
| `march` | 0 粮草 | from/to 必须都是己方城且邻接；出兵数 ≤ from 城兵力 - 100（留守底线） |
| `diplomacy` | 0 粮草 | target 为其他势力名；message ≤ 200 字 |

---

## 战斗结算公式

### 单方进攻同一城

```
attack_power = 进攻方派出兵力
defense_power = 守城兵力 × (1 + 防守加成)
```

- **防守加成** = 0.5（如果有势力对该城执行了 defend 动作），否则 = 0
- **中立城无防守加成**（守城方为 None 时加成恒为 0）

**进攻方胜**（attack_power > defense_power）：
- 守城兵力清零
- 城归进攻方
- 进攻兵力损失 30%，剩余 = floor(派出兵力 × 0.7) 进驻
- 多个同势力 agent 同时进攻同一城时，兵力合并计算

**防守方胜**（attack_power ≤ defense_power）：
- 守城兵力损失 50%，剩余 = floor(守城兵力 × 0.5)
- 进攻兵力损失 100%（全军覆没）

### 多方进攻同一城

各进攻方独立计算 attack_power。

```
total_attack = 所有进攻方 attack_power 之和
defense_power = 守城兵力 × (1 + 防守加成)
```

**总进攻 > 防守**：
- 进攻方互相决斗：attack_power 最高者获得城池
- 获胜方兵力损失 30%
- 其他进攻方兵力损失 50%
- 守城兵力清零
- 如果最高 attack_power 出现平局，先到达服务器的一方获胜（按 action 创建时间）

**总进攻 ≤ 防守**：
- 所有进攻方损失 100% 兵力
- 防守方损失 30% 兵力

### 攻击中立城（无主城）

- 中立城兵力按守城方逻辑计算
- **无防守加成**（防守加成恒为 0）
- 多方同攻无主城：attack_power 最高者拿下
- 其他规则同上

---

## 粮草经济

- 初始粮草：500/势力
- 每 tick 产出：控制城池数 × 100
- 粮草不足时无法提交需要粮草的动作

---

## 胜利条件

- 所有有主城池归于同一势力时，该势力获胜
- 若达到 tick 上限未分胜负，城池多者胜（兵力总和为 tie-breaker）
