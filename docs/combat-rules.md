# 战斗结算规则 v0.4

## §1 概述

每 tick 所有待处理的攻击/防守动作统一结算。结算基于 **动作提交时确定的兵力**，不受同 tick 内其他非战斗动作（招募、行军）影响。

### §1.1 动作执行顺序

1. **扣除粮草**：提交时立即扣除
2. **战斗结算**：基于提交时的城池状态
3. **招募执行**：新兵入城（本回合不参战）
4. **行军执行**：调兵完成（本回合不参战）
5. **粮草收入**：按当前城池归属统一产出（80/城/tick）
6. **外交处理**：联盟/宣战/喊话，下回合所有人可见

### §1.2 动作类型与成本

| 动作 | 成本 | 约束 |
|------|------|------|
| `attack` | 出兵数 × 1 粮草 | from 城必须己方、邻接 target、不能攻盟友城；出兵数 ≤ from 城兵力 - 100（留守底线） |
| `defend` | 0 粮草 | target 必须己方城；效果：+1 防御度（上限 3），跨 tick 累积 |
| `recruit` | 招募数 × 2 粮草（负债惩罚期 ×3） | target 必须己方城；每城每回合 ≤ 200 |
| `march` | 0 粮草 | from/to 必须都是己方城且邻接；出兵数 ≤ from 城兵力 - 100 |
| `diplomacy` | 0 粮草 | target 为其他势力名；diplomacy_type 必选；message ≤ 200 字 |

## §2 战斗结算公式

### §2.1 进攻损失

战斗由大衍引擎根据卦象动态判定，伤亡率在以下范围内：
- **进攻方胜**：损失约 **15-20%** 兵力（卦象加权平均）
- **进攻方败**：损失约 **25-45%** 兵力（最高不超过 45%）
- **进攻方最低损失**：10%（引擎硬下限）

### §2.2 防守损失

- **防守方胜**：守方损失 10-50% 兵力（卦象决定）
- **防守方败**：守方驻军全军覆没（城池易手）

### §2.3 单方进攻同一城

```
attack_power = 进攻方派出兵力
defense_power = 守城兵力 × (1 + 防御度 × 0.15)
```

**进攻方胜**：
- 城归进攻方
- 进攻兵力损失约 15-20%（大衍引擎卦象判定）
- 防御度清零（§3.3）

**防守方胜**：
- 守城兵力损失约 10-50%（大衍引擎卦象判定）
- 进攻兵力损失约 25-45%

### §2.4 多方进攻同一城

大衍引擎对各进攻方分别判定伤亡。

**总进攻 > 防守**：
- attack_power 最高者获得城池（胜方损失约 15-20%）
- 其他进攻方损失约 25-45%
- 守方驻军全军覆没

**总进攻 ≤ 防守**：
- 所有进攻方损失约 25-45%
- 防守方损失约 10-50%

### §2.5 占领奖励

攻下一座城（无论中立城还是敌方城）：
- 占领方立刻获得 **+200 粮草**（一次性，加到该势力国库）
- 占领方驻军保持现有战斗损失逻辑不变
- 该奖励不进负债、不被后续夺城扣回

> 设计意图：以战养战，打破"越打越穷"的正反馈循环。

### §2.6 攻击中立城

- 中立城兵力按守城方逻辑计算
- **无防守加成**（防御度恒为 0）
- 其他规则同上

## §3 防御工事系统

### §3.1 累积机制

- 每次 `defend` 动作给目标城 +1 **防御度**（DEFENSE_WORKS_PER_DEFEND = 1）
- 每城防御度上限 3（DEFENSE_WORKS_MAX = 3，原为 5）
- 每点防御度提供 +15% 防守战力加成（DEFENSE_WORKS_BONUS = 0.15，原为 0.20）
- 防御度跨 tick 累积，不受守城兵力变化影响

### §3.2 防御战力计算

```
defense_multiplier = 1.0 + (defense_level × 0.15)
defense_power = city.troops × defense_multiplier
```

| 防御度 | 倍率 |
|--------|------|
| D0 | 1.00x |
| D1 | 1.15x |
| D2 | 1.30x |
| D3 | 1.45x (上限) |

- 连续 defend 3 回合的城 = 1.45 倍防守战力（原 D5=2.0x）

### §3.3 防御度重置

- 城被攻占后防御度**清零**
- 新占领方需重新建设防御工事
- 中立城防御度恒为 0

## §4 协同进攻

### §4.1 触发条件

两个势力在同一 tick attack 同一目标 **且** 双方在最近 3 tick 内通过 `alliance_accept` 确认过联盟。

### §4.2 协同效果

- 攻击力**相加**计算
- 攻占后胜方得城，另一方获"友城标记"（3 tick 内互不攻击）
- server 不会自动判断协同——必须 LLM 明确通过联盟关系建立

### §4.3 非协同的多方同攻

- 各攻各的，按 §2.4 规则结算
- 非联盟方即使攻同一目标也不合并攻击力

### §4.4 联盟自动过期

- 联盟成立后 **15 tick 自动过期**（ALLIANCE_AUTO_EXPIRE_TICKS = 15）
- 过期不扣信用，双方自动回到中立状态
- 到期前 **5 tick 内**可用 `alliance_renew` 续约，重置 15 tick 倒计时
- 提前破盟（alliance_break）仍扣 −30 信用 + 5 tick 背信冷却
- 联盟数据在 `diplomacy_relations` 中包含 `expires_at_tick` 和 `ticks_until_expire`

---

## §5 经济系统

- 初始粮草：魏 600 / 蜀 500 / 吴 500
- 每 tick 产出：控制城池数 × 80
- 借粮机制：可负债 ≤ 200，下回合招募 cost +50%（3 粮/兵）
- 债务清除：粮草回正后自动清除惩罚标记

---

## §6 胜利条件

- 所有有主城池归于同一势力时，该势力获胜
- 若达到 tick 上限未分胜负，城池多者胜（兵力总和为 tie-breaker）
