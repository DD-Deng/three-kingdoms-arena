# v0.8 — Lobby v2 + 观战页 v2

**Date**: 2026-05-17

## 技术栈升级

- **从 Babel standalone → Vite + React 19**：移除浏览器内编译，改为 build-time 编译
- 新子项目 `frontend-v2/`（`Vite 6 + React 19 + React Router 7`）
- Docker 多阶段构建（`node:20-alpine` build → `python:3.11-slim` serve）
- 部署于 `/v2/` 路径，`/lobby` 自动重定向至 `/v2/lobby`

## Lobby v2 完整重做

**五种槽位状态**:
- **空缺**：灰框，显示阵营名+君主名，可「配 AI 托管」或「加入」
- **AI 占用**：金框，显示 AI 标识，真人可「抢占」
- **已占用**：淡底，显示 IP 末段，「未就绪」标签
- **已就绪**：绿框，阵营色饱和，「已就绪」勾选
- **倒计时/对局中**：金框锁定，阵营+君主+agent 名

**新增功能**:
- 配 AI 托管：任意空缺位可一键配 AI，AI 自动就绪
- 抢位：倒计时未启动前，真人可抢占 AI 占位（AI 让出）
- 释放 AI：倒计时未启动前，可释放 AI 托管
- 5 秒倒计时全屏浮层
- 乐观更新：加入/配 AI 后立即更新本地 UI

## 观战页 v2 完整重做

**布局**（参考 FM 叙事中心 + Lichess）:
- 左栏 55%：SVG 7 城抽象地图 + Eval Bar 实力条 + 阵营信息卡 + 胜率曲线
- 右栏 45%：评书叙事流
- 底部：事件 feed（可展开战斗报告）+ Tick 进度条

**新增**:
- SVG 7 城地图，城池归属颜色，闪烁攻占动画
- 三色实力条（蜀绿/魏蓝/吴红）
- 胜率曲线（前端累积每 tick 实力值）
- 事件流，点击展开完整战报（含大衍卦象）
- 评书增量叙事：每 5 tick 自动生成一段

## API 新增

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/lobby/assign-ai` | POST | 配 AI 托管到指定槽位 |
| `/v1/lobby/release-ai` | POST | 释放 AI 或玩家自己槽位 |
| `/v1/lobby/status` | GET | 新增 `chapters` 字段 |

**新增 error codes**:
- `COUNTDOWN_STARTED`：倒计时已启动，无法加入

## 数据模型

- `Game.chapters`：JSON 数组，每 5 tick 增量评书段落
- `Slot.status`：新增 `ai_managed` 状态

## 已知限制

- 旧 `static/` 前端保留为 fallback（待 v0.9 删除）
- `/current-game` 与 `/v1/lobby/status` 可能返回不同 game（不同视图）
- 槽位断线宽容期 600s，快速反复加入需等待
