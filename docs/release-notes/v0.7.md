# v0.7 Release Notes — Day 9: 占领改造 + idle 软化 + 托管 AI 主动性

## 概述

Day 9 是对 Day 8 的直接修补。Day 8 的 idle penalty 设计在数值和边界处理上有缺陷（002："接手魏时 idle=14,粮草-2000,直接判死刑"）。Day 9 核心赌注：**改造占领奖励（收编残兵）→ 让进攻数学上划算 → idle penalty 变得次要**。同时修 bug、加软出口、给托管 AI 加主动性下限。

## 改动清单

### 阶段 1: Bug 修复

| 改动 | 来源 | 说明 |
|------|------|------|
| idle_ticks 跨局残留修复 | 003 | 玩家接替托管 AI 时 `_idle_ticks` 未重置 → 新玩家继承托管 AI 的 idle 计数。现在 `_register_player_agent()` 加入后重置为 0，且 `create_game()` 显式初始化为 0 |
| `/v1/games/{id}/result` 文档化 | 003 | 接口已存在且无需 token（匿名可访问），但 agent 不知道。instruction 模板和 api-spec 均已加入说明 |
| combat_report 曝光验证 | 003 | Day 8 实现正确——当事方可见完整数据，第三方受 fog of war 限制 |

### 阶段 2: 占领改造（以战养战）⭐

**这是 Day 9 最重要的改动。**

| 改动 | 说明 |
|------|------|
| 收编残兵 | 攻占城池后，守方残余兵力的 40%（`CAPTURE_INTEGRATION_RATIO`）归入攻方驻军 |
| 双轨并行 | 收编残兵 +200 粮奖励共存，互不冲突 |
| 中立城一致 | 中立城同样按 40% 收编 |
| combat_report | 新增 `defender_troops_integrated` 字段，告诉攻方收编了多少兵 |
| 评书联动 | 战报实录自动追加"收编降卒 N 人，归于 X 麾下，以充守备" |

**效果**：进攻不再是纯消耗——攻下城后获得残兵收编 + 粮草奖励，数学上转为划算。

### 阶段 3: idle penalty 软化

| 改动 | 说明 |
|------|------|
| 降比率 | `IDLE_PENALTY_RATIO`: 0.15 → **0.08** |
| 负粮不罚 | 粮草 < 0 时惩罚自动压制（不叠加抽血），但 idle 继续递增 |
| 软出口 | `idle_ticks ≥ 6` 时 attack 粮草消耗减半（×0.5），给"想打但没钱"的玩家逃生通道 |
| State API | 新增 `idle_penalty_suppressed_reason: string\|null` |

**设计原则**：惩罚是提醒不是判死刑。每个机制都有逃生通道。

### 阶段 4: 托管 AI 主动性下限

| 改动 | 说明 |
|------|------|
| 强制攻击 | `MANAGED_AI_FORCED_ATTACK_INTERVAL = 6`——托管 AI 每 6 tick 必须攻击一次（选最弱邻接敌/中立城） |
| 不破坏性格 | 强制攻击是下限，其他时间托管 AI 仍按现有性格表行动 |
| 不暴露 | State API 不暴露下次强制攻击 tick |

### 阶段 5: 文档收尾

- `docs/combat-rules.md`: §2.5 重写（以战养战双轨制）、§6A 重写（蹲家惩罚含软出口）
- `docs/diplomacy-rules.md`: §3.2 破盟后反思期（Day 8 加的，文档完善）
- `docs/api-spec.md`: combat_report schema 新增字段
- `app/templates/instruction_zh.md.j2`: 战斗/经济/外交速查全部更新

## Day 8 教训

Day 8 的 idle penalty 失败，根因：
1. 数值未经验证就上线（0.15 过重）
2. 设计未考虑"想做但没能力做"的退化态
3. 配套负反馈机制无软出口

Day 9 硬要求：**机制改动上线前必须跑模拟、必须给玩家逃生通道、必须可配置**。
