- `git ls-files logs/`
- `git ls-files .arena_credentials/`
- `git ls-files | grep -E "\.env|secret|credential|api[_-]?key" -i`

1. 把它们加入 .gitignore

我特意做的几个设计
1. 分 4 阶段而不是一口气干完
Claude Code 一次性改太多文件容易失控。分阶段让你有 checkpoint 可以审查,出问题能立刻回滚。
2. 阶段 1 是安全检查
万一你之前真的 push 了 API key,这事比写 README 紧急 100 倍。所以放最前面,先排雷。
3. 阶段 2 让 Claude Code 给你文案,不让它直接改
GitHub About / Topics / Description 这些只能在网页填,Claude Code 改不了。但它可以给你写好文案,你复制粘贴 5 分钟搞定。
4. README 重写部分给了详细结构和内容要点
避免 Claude Code 自由发挥写一份太"AI 味"的 README。我把关键内容(7 城、5 动作、评书等)明确列出来,它就会精准复用你的产品现状。
5. 阶段 4 是诊断不是改动
GitHub Pages 现状我没看到,贸然让 Claude Code 改容易出错。先让它报告现状,你确认了我们再决定下一步。
